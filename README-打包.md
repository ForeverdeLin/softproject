# 打包说明

## 📦 快速打包

### 服务器打包

1. **双击运行** `打包服务器.bat`
2. 等待打包完成
3. EXE文件位于：`dist\失物招领系统-服务器.exe`

### 客户端打包

1. **双击运行** `打包客户端.bat`
2. 等待打包完成
3. EXE文件位于：`dist\失物招领系统-客户端.exe`

## 🚀 使用方式

### 服务器

1. 双击运行 `失物招领系统-服务器.exe`
2. 命令行窗口会显示服务器信息
3. **保持窗口打开**，不要关闭
4. 服务器运行在：`http://0.0.0.0:5000`

### 客户端

1. **确保服务器已启动**
2. 双击运行 `失物招领系统-客户端.exe`
3. **浏览器会自动打开**客户端界面
4. 注册账号、登录、发布失物
5. 多个客户端可以共享数据

## 📝 多客户端使用场景

### 场景1：局域网使用

**服务器电脑（IP: 10.161.37.34）**
- 运行：`失物招领系统-服务器.exe`
- 服务器运行在：`http://10.161.37.34:5000`

**客户端电脑1**
- 运行：`失物招领系统-客户端.exe`
- 自动连接到：`http://10.161.37.34:5000`（已配置）
- 浏览器自动打开：`http://localhost:5001`

**客户端电脑2**
- 运行：`失物招领系统-客户端.exe`
- 需要修改服务器地址（如果不同）
- 浏览器自动打开：`http://localhost:5001`

**数据共享：**
- 客户端1发布失物 → 服务器保存
- 客户端2刷新页面 → 可以看到客户端1的数据

## ⚙️ 配置服务器地址

如果服务器IP地址变化，需要修改客户端配置：

**方法1：修改配置文件后重新打包**
编辑 `app/config.py`：
```python
SERVER_URL = 'http://新的服务器IP:5000'
```
然后重新运行 `打包客户端.bat`

**方法2：使用环境变量（不推荐，需要手动设置）**

## 🔧 故障排查

### 服务器无法启动
- 检查端口5000是否被占用
- 检查防火墙设置
- 查看 `error.log` 文件

### 客户端无法连接服务器
- 检查服务器是否已启动
- 检查服务器IP地址是否正确
- 检查防火墙设置
- 确保在同一网络

### 浏览器没有自动打开
- 手动打开浏览器访问：`http://localhost:5001`
- 检查是否有其他程序占用5001端口
