# 软件工程期末大作业 - 选题与需求分析

## 一、选题确定

### 项目名称
**校园失物招领智能匹配系统**

### 项目类型
基于规则型智能体的失物招领平台

### 选择理由
1. **智能体实现简单**：采用规则型智能体，满足≥3种决策分支的要求
2. **业务逻辑清晰**：失物发布、招领发布、智能匹配、用户交互
3. **数据库设计简单**：用户表、失物表、招领表、匹配记录表
4. **技术栈灵活**：Python + Web框架 + SQLite/MySQL
5. **演示效果好**：智能体决策过程可视化，匹配结果直观

## 二、需求分析

### 2.1 功能需求

#### 2.1.1 用户管理模块
- **用户注册**：学号、姓名、联系方式、邮箱
- **用户登录**：账号密码验证
- **个人信息管理**：修改个人信息、查看历史记录

#### 2.1.2 失物发布模块
- **发布失物信息**：
  - 物品名称、类别（钱包/手机/书籍/其他）
  - 丢失地点、丢失时间
  - 物品特征描述（颜色、品牌、型号等）
  - 上传图片（可选）
  - 联系方式

#### 2.1.3 招领发布模块
- **发布招领信息**：
  - 物品名称、类别
  - 拾获地点、拾获时间
  - 物品特征描述
  - 上传图片（可选）
  - 联系方式

#### 2.1.4 **智能体匹配模块（核心功能）**
- **感知阶段**：
  - 读取新发布的失物/招领信息
  - 从数据库获取历史数据
  - 提取关键特征（类别、地点、时间、颜色等）

- **决策阶段**（规则型智能体，≥3种决策分支）：
  1. **类别匹配规则**：物品类别必须相同
  2. **地点相似度规则**：丢失地点与拾获地点距离判断（同一区域/相邻区域/不同区域）
  3. **时间匹配规则**：丢失时间与拾获时间的时间差判断（24小时内/3天内/一周内/超过一周）
  4. **特征相似度规则**：颜色、品牌等特征关键词匹配
  5. **综合评分规则**：综合以上规则计算匹配度分数

- **执行阶段**：
  - 生成匹配结果列表（按匹配度排序）
  - 自动发送通知给失主和拾获者
  - 记录匹配历史

#### 2.1.5 匹配结果展示模块
- 显示匹配度分数
- 显示匹配原因（哪些规则匹配成功）
- 提供联系方式查看
- 标记已解决/未解决

#### 2.1.6 搜索查询模块
- 按类别搜索
- 按地点搜索
- 按时间范围搜索
- 关键词搜索

### 2.2 非功能需求

#### 2.2.1 性能需求
- 匹配响应时间 < 2秒
- 支持并发用户数 ≥ 50
- 数据库查询优化

#### 2.2.2 安全需求
- 用户密码加密存储
- 防止SQL注入
- 防止XSS攻击
- 用户身份验证

#### 2.2.3 可用性需求
- 界面简洁直观
- 操作流程清晰
- 支持移动端访问（响应式设计）

#### 2.2.4 可维护性需求
- 代码规范，面向对象设计
- 模块化设计，易于扩展
- 完整的文档和注释

### 2.3 智能体详细设计

#### 2.3.1 智能体类型
**规则型智能体（Rule-Based Agent）**

#### 2.3.2 感知-决策-执行循环

**感知（Perception）**：
- 输入：新发布的失物信息或招领信息
- 数据获取：从数据库读取所有相关的招领/失物记录
- 特征提取：类别、地点、时间、颜色、品牌等

**决策（Decision）**：
- 规则1：类别匹配（必须相同）
- 规则2：地点相似度计算（距离判断）
- 规则3：时间匹配度计算（时间差判断）
- 规则4：特征相似度计算（关键词匹配）
- 规则5：综合评分计算（加权求和）

**执行（Action）**：
- 输出匹配结果列表
- 发送通知消息
- 更新数据库记录

#### 2.3.3 决策规则详细设计

```
匹配度计算公式：
总分 = 类别匹配分(40分) + 地点相似度(25分) + 时间匹配度(20分) + 特征相似度(15分)

规则1：类别匹配（40分）
- 类别相同：40分
- 类别不同：0分（直接排除）

规则2：地点相似度（25分）
- 同一地点：25分
- 相邻区域（如：同一教学楼不同楼层）：20分
- 同一校区不同区域：15分
- 不同校区：5分

规则3：时间匹配度（20分）
- 24小时内：20分
- 3天内：15分
- 一周内：10分
- 超过一周：5分

规则4：特征相似度（15分）
- 颜色匹配：5分
- 品牌匹配：5分
- 其他特征关键词匹配：5分（最多5分）

规则5：综合判断
- 总分 ≥ 60分：高匹配度（推荐）
- 总分 40-59分：中等匹配度（可能）
- 总分 < 40分：低匹配度（不推荐）
```

## 三、技术选型

### 3.1 开发语言
**Python 3.8+**
- 纯面向对象编程
- 丰富的库支持
- 易于实现智能体逻辑

### 3.2 Web框架
**Flask** 或 **Django**
- Flask：轻量级，适合快速开发
- Django：功能完整，自带ORM

### 3.3 数据库
**SQLite**（开发）或 **MySQL**（生产）
- SQLite：无需安装，适合演示
- MySQL：生产环境使用

### 3.4 前端技术
- HTML5 + CSS3 + JavaScript
- Bootstrap（响应式UI框架）
- 可选：Vue.js（如果需要更复杂交互）

### 3.5 部署方案
**Docker + Docker Compose**
- 一键启动
- 包含Web服务、数据库、依赖
